# Pitch Deck Storytelling

Transform pitch decks from information dumps into compelling narratives. This skill ensures every slide advances a story that investors can absorb whether they spend 30 seconds or 30 minutes with your deck.

## Core Philosophy

**The deck should work at three levels:**

1. **Headers only** — Reading just the headlines tells a complete, coherent story
2. **Skim mode** — Headers + visuals convey all key points
3. **Deep read** — Full content provides complete details and evidence

This allows investors to engage at whatever depth they choose while still absorbing the core narrative.

---

## Principle 1: Section Dividers as Narrative Signposts

**Purpose:** Break the deck into clear chapters that create mental breathing room and anticipation.

### Implementation

Create section title slides that:
- Use a contrasting background (dark on light deck, or vice versa)
- Display only the section name in large, centered text
- Include a section number indicator (e.g., "1 / 6")
- Add subtle decorative elements (lines, shapes) for polish
- Contain NO detailed content — just the chapter title

### Standard Section Structure for Investor Decks

```
1. THE PROBLEM      — Why this matters
2. THE SOLUTION     — What you've built
3. TRACTION         — Proof it works
4. THE BIGGER OPPORTUNITY — Platform vision
5. THE BUSINESS     — How you make money
6. TEAM & ASK       — Who you are, what you need
```

### Why It Works

Section slides:
- Give the audience mental breaks
- Signal transitions so investors can recalibrate
- Create anticipation for what's coming
- Make the deck feel organized and professional
- Allow investors to jump to sections of interest

## Principle 2: Story-Driven Headers (Not Labels)

**Transform:** Generic labels → Narrative headlines that make claims

### The Before/After Test

| ❌ Label | ✅ Story Header |
|----------|-----------------|
| Platform Overview | One pilot revealed something bigger. |
| Market Opportunity | 60% still use spreadsheets. |
| Expansion Strategy | The same sale opens multiple doors. |
| Competitive Moat | Every deployment makes us harder to replace. |
| Traction | Product-market fit: validated. |
| Team | Built by operators who lived this problem. |

### The Rules

1. **Each header is a complete sentence** — It makes a claim, not a label
2. **Headers end with periods** — Signals assertion, not placeholder
3. **The header alone should be interesting** — No context needed
4. **Every header advances the narrative** — Cut filler slides

### Header + Subtitle Pattern

```
Main Header:    "One pilot revealed something bigger."
                (32pt, bold, primary color)

Subtitle:       "The same AI engine powers multiple billion-dollar verticals."
                (14pt, italic, accent color)
```

The subtitle explains or expands the header's promise without duplicating it.

## Principle 3: Headlines Tell a Complete Story

**Test:** Read ONLY the slide headers in sequence. Do they tell a coherent story?

### Example Narrative Arc (23 slides)

```
1. SCRIBE
2. Case workers are drowning.
3. THE PROBLEM
4. 60% still use spreadsheets.
5. Documentation should start with conversation.
6. THE SOLUTION
7. Scribe: Document once, use everywhere.
8. The timing has never been better.
9. The competition is broken.
10. Bonterra promised AI—then delivered vaporware.
11. TRACTION
12. Product-market fit: validated.
13. THE BIGGER OPPORTUNITY
14. One pilot revealed something bigger.
15. The same sale opens multiple doors.
16. Every deployment makes us harder to replace.
17. THE BUSINESS
18. A model that scales.
19. The path to profitability is clear.
20. TEAM & ASK
21. Built by operators who lived this problem.
22. We're raising $650K to accelerate.
23. SCRIBE
```

### The Arc Structure

```
Hook → Problem → Insight → Solution → Proof → Vision → Model → Team → Ask → Close
```

Each section transitions naturally to the next:
- Problem creates urgency
- Solution addresses problem
- Proof validates solution
- Vision expands opportunity
- Model shows sustainability
- Team builds confidence
- Ask enables action

## Principle 4: Visual Hierarchy Supports Story

### Section Slides (Chapter Breaks)
- **Background:** Full dark/contrasting color
- **Text:** Minimal — just section title
- **Contrast:** High (white on dark)
- **Purpose:** Visual pause, mental reset

### Content Slides
- **Background:** Clean (white or very light)
- **Header:** Story-driven, top of slide
- **Subtitle:** Explanatory, below header
- **Content:** Visual evidence below
- **Footer (optional):** Key insight or talking point

### Platform/Strategy Slides
```
┌─────────────────────────────────────┐
│ Story-driven header (tells the claim)│
│ Subtitle (explains the mechanism)    │
├─────────────────────────────────────┤
│                                     │
│   [ Visual content / diagram ]      │
│                                     │
├─────────────────────────────────────┤
│ "Investor talking point in quotes"  │
└─────────────────────────────────────┘
```

## Principle 5: Investor Talking Points

**Purpose:** Pre-write the language investors will use when describing your company to others.

### Implementation

On key slides, include quotable soundbites in a distinct visual treatment:

```
┌────────────────────────────────────────────────────┐
│ "We're building the compliance-aware AI layer     │
│  for operations-heavy organizations."             │
└────────────────────────────────────────────────────┘
```

### Characteristics of Good Talking Points

- **Self-contained** — Makes sense without context
- **Quotable** — Reads naturally as a verbal statement
- **Memorable** — Uses concrete language, not jargon
- **Differentiating** — Captures what makes you unique
- **Repeatable** — Something an investor would actually say

### Where to Place Them

- Moat/defensibility slides
- Platform/vision slides
- Business model slides
- After key proof points

### Examples

```
"The same conversation that landed our nonprofit pilot 
 opened the door to 400-store retail."

"Nonprofit beachhead proves unit economics. 
 Enterprise is pure margin expansion."

"We're not building a nonprofit tool—we're building 
 the AI layer for operations-heavy organizations."
```

## Principle 6: Narrative Transitions

### Bridge Sections with Teasers

Before a major section (especially the "vision" section), add narrative tension:

```
┌─────────────────────────────────────────────────────┐
│              THE BIGGER OPPORTUNITY                 │
│                                                     │
│  "Our discovery call revealed we're not just       │
│   building a nonprofit tool.                        │
│                                                     │
│   We're building an AI platform for                │
│   operations-heavy organizations."                  │
└─────────────────────────────────────────────────────┘
```

### PMF Slide as Transition

Position your proof-of-concept / traction slide as a bridge:

```
Header: "Product-market fit: validated."

Content: [proof points, metrics, quotes]

Transition line at bottom:
"What happened next surprised us."
```

This creates anticipation for the platform expansion section.

## Application Checklist

When creating or reviewing a pitch deck:

### Structure
- [ ] Deck has 5-7 clear sections with title slides
- [ ] Section order follows logical narrative arc
- [ ] Total slide count is 15-25 (excluding appendix)

### Headers
- [ ] Every header makes a claim (is a complete sentence)
- [ ] Headers work standalone as a story sequence
- [ ] No generic labels (Overview, Summary, Details)
- [ ] Subtitles expand, don't repeat, the header

### Visual Hierarchy
- [ ] Section slides are visually distinct from content
- [ ] Content slides have consistent header/subtitle pattern
- [ ] Key slides include quotable talking points
- [ ] White space creates breathing room

### Story Test
- [ ] Read only the headers — does it tell a story?
- [ ] Could an investor understand the pitch at a glance?
- [ ] Does the narrative build to the ask?
- [ ] Are transitions between sections clear?

### Investor Utility
- [ ] Key slides have pre-written talking points
- [ ] Metrics are prominent and scannable
- [ ] Proof points lead naturally to expansion
- [ ] The ask is specific and justified

## Technical Implementation (pptxgenjs)

### Section Title Slide Template

```javascript
function createSectionSlide(pres, sectionName, sectionNumber, totalSections) {
  const COLORS = {
    darkTeal: "134E4A",
    lightTeal: "14B8A6",
    white: "FFFFFF"
  };
  
  let slide = pres.addSlide();
  slide.background = { color: COLORS.darkTeal };
  
  // Section number indicator
  slide.addText(`${sectionNumber} / ${totalSections}`, {
    x: 0.5, y: 0.4, w: 1.5, h: 0.3,
    fontSize: 12, fontFace: "Arial",
    color: COLORS.lightTeal
  });
  
  // Main section title
  slide.addText(sectionName, {
    x: 0.5, y: 2.2, w: 9, h: 1.2,
    fontSize: 48, fontFace: "Arial Black", bold: true,
    color: COLORS.white, align: "center", valign: "middle"
  });
  
  // Decorative line
  slide.addShape(pres.shapes.RECTANGLE, {
    x: 4, y: 3.5, w: 2, h: 0.05,
    fill: { color: COLORS.lightTeal }
  });
  
  return slide;
}
```

### Story-Driven Content Slide Template

```javascript
function createStorySlide(pres, header, subtitle, addContent) {
  const COLORS = {
    darkText: "1E293B",
    teal: "0D9488",
    white: "FFFFFF"
  };
  
  let slide = pres.addSlide();
  slide.background = { color: COLORS.white };
  
  // Story-driven header
  slide.addText(header, {
    x: 0.5, y: 0.4, w: 9, h: 0.5,
    fontSize: 32, fontFace: "Arial Black", bold: true,
    color: COLORS.darkText
  });
  
  // Explanatory subtitle
  slide.addText(subtitle, {
    x: 0.5, y: 0.9, w: 9, h: 0.3,
    fontSize: 14, fontFace: "Arial", italic: true,
    color: COLORS.teal
  });
  
  // Custom content callback
  if (addContent) {
    addContent(slide);
  }
  
  return slide;
}
```

### Talking Point Box Template

```javascript
function addTalkingPoint(slide, quote, x, y, width, height) {
  const COLORS = {
    lightGray: "F1F5F9",
    teal: "0D9488",
    darkText: "1E293B"
  };
  
  // Background box
  slide.addShape(pres.shapes.RECTANGLE, {
    x: x, y: y, w: width, h: height,
    fill: { color: COLORS.lightGray }
  });
  
  // Left accent bar
  slide.addShape(pres.shapes.RECTANGLE, {
    x: x, y: y, w: 0.06, h: height,
    fill: { color: COLORS.teal }
  });
  
  // Quote text
  slide.addText(`"${quote}"`, {
    x: x + 0.15, y: y + 0.05, w: width - 0.25, h: height - 0.1,
    fontSize: 10, fontFace: "Arial", italic: true,
    color: COLORS.darkText, valign: "top"
  });
}
```

## Quick Reference

| Principle | Implementation | Test |
|-----------|----------------|------|
| Section Dividers | Dark background, centered title, section number | Does it create a visual pause? |
| Story Headers | Complete sentence ending in period | Could this headline run in a newspaper? |
| Complete Story | Headers read as narrative sequence | Does reading only headers tell the story? |
| Visual Hierarchy | Section/content contrast, consistent patterns | Can you identify slide type instantly? |
| Talking Points | Quoted soundbites in distinct boxes | Would an investor repeat this phrase? |
| Transitions | Bridge content between sections | Does each section flow to the next? |

## When to Use This Skill

**Use for:**
- Investor pitch decks (seed, Series A, etc.)
- Sales presentations to enterprise buyers
- Board presentations requiring narrative flow
- Internal strategy presentations
- Demo day presentations

**Adapt for:**
- Product demos (more visual, less talking points)
- Technical presentations (keep headers, adjust content depth)
- One-pagers (extract key headers and talking points)

---

## Common Mistakes to Avoid

1. **Label headers** — "Market Size" instead of "The market is massive and underserved."
2. **Missing section dividers** — Deck feels like one long wall of content
3. **Talking at investors** — No pre-written language for them to use
4. **No narrative arc** — Slides could be reordered without impact
5. **Visual inconsistency** — Every slide uses different layout
6. **Too much text** — Forces deep read, breaks skim mode
7. **Weak transitions** — Sections feel disconnected
8. **Buried ask** — Not clear what you want and why
